# Ansible Configuration — k3s Cluster

This directory contains Ansible playbooks and inventory for bootstrapping and managing the k3s cluster.

## Inventory

The inventory file (`inventory/hosts.ini`) is **auto-generated by Terraform**. Do not edit it manually.

After running `terraform apply` in `../terraform/proxmox/`, the inventory will be created at:
```
infra/ansible/inventory/hosts.ini
```

### Inventory Structure

The generated INI inventory includes:

- `[k3s_servers]` — All 3 k3s nodes
- `[k3s_first_node]` — First node (for initial cluster bootstrap)
- `[k3s_other_nodes]` — Remaining nodes (for joining existing cluster)

### Connection Variables

These are set automatically via `[k3s_servers:vars]`:

| Variable | Value | Source |
|----------|-------|--------|
| `ansible_user` | `ansible` | `var.ansible_user` in Terraform |
| `ansible_ssh_private_key_file` | `~/.ssh/id_ed25519` | `var.private_key_path` in Terraform |
| `ansible_python_interpreter` | `/usr/bin/python3` | Hardcoded (Debian default) |

## Next Steps

Once VMs are running and `terraform apply` has generated the inventory:

1. **Verify connectivity:**
   ```bash
   ansible -i inventory/hosts.ini k3s_servers -m ping
   ```

2. **Bootstrap k3s cluster:**
   ```bash
   ansible-playbook -i inventory/hosts.ini playbooks/k3s-init.yml
   ```

3. **Configure NFS client:**
   ```bash
   ansible-playbook -i inventory/hosts.ini playbooks/nfs-client.yml
   ```

## Playbooks

| Playbook | Purpose | When to Run |
|----------|---------|-------------|
| `k3s-init.yml` | Bootstrap 3-node HA k3s cluster | After VM creation |
| `nfs-client.yml` | Configure NFS mount for shared storage | After k3s init |
| `k3s-upgrade.yml` | Rolling upgrade of k3s version | When upgrading k3s |

## Regenerating Inventory

If VM IPs change or you modify the k3s_nodes map in Terraform:

```bash
cd ../terraform/proxmox
tofu apply
# This will regenerate inventory/hosts.ini
```
