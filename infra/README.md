# infra/

Infrastructure-as-code for the homelab deployment of the eve-market-analytics
platform. The checked-in repo currently covers Proxmox VM provisioning, Ansible-based
k3s bootstrap, the external PostgreSQL VM for Airflow and MLflow metadata, shared NFS
storage wiring, and the monitoring stack. PostgreSQL runs on its own Proxmox VM
outside the cluster, not as another checked-in Kubernetes service.

## Directory Structure

```text
infra/
├── Makefile                    # Single entrypoint for infrastructure workflows
├── terraform/
│   └── proxmox/                # Layer 1: VM provisioning + inventory + secret generation
├── ansible/                    # Layer 2: OS config + k3s bootstrap + PostgreSQL + NFS client setup
├── helm/                       # Layer 3: Helm values overrides committed today
│   ├── grafana.yml
│   └── victoriametrics.yml
├── k8s/                        # Layer 3: Raw manifests plus Kustomize dashboard sources
│   └── monitoring/             # Grafana dashboard JSON, kustomization, and rendered JSON
└── third_party/                # Scoped licenses and provenance for vendored assets
```

## Bootstrap Order

Apply infrastructure in this sequence:

```text
1. OpenTofu     (terraform/proxmox) -> VMs created, inventory generated, DB secrets generated
2. Ansible      (ansible/)          -> k3s bootstrapped, NFS mounted, PostgreSQL configured, kubeconfig copied
3. kubectl/Helm (k8s/ + helm/)      -> base resources and Kubernetes-managed services deployed
```

`make bootstrap` exists for the eventual full stack, but the currently committed repo
does not yet include a complete end-to-end application deployment. For the current repo
state, use the layered workflow below instead of assuming every Kubernetes-managed
service is ready to deploy.

The repo uses both local `pre-commit` hooks and a GitHub Actions workflow at
`.github/workflows/infra-checks.yml` for infrastructure validation. `pre-commit`

### Optional CI/CD Runners

The current shared CI/CD setup runs on GitHub Actions self-hosted runners with
labels `[self-hosted, Linux, X64, eve, ci, python]`.

Current layout:

- 3 Debian 13 LXCs
- 1 GitHub Actions runner per LXC

This runner fleet is currently used by the repo, but it is optional. You can run
the same checks locally with `make pre-commit-run` or `make validate`.

### Proxmox Cluster

1. Create a Proxmox API token at Datacenter -> Permissions -> API Tokens.
2. Prepare a Debian 13 cloud-init template.
3. Use a shared datastore that supports imports and snippets.
4. Ensure `vmbr0` or your chosen bridge is available.
5. Keep SSH access available for snippet upload.

### TrueNAS NFS Share

Shared NFS storage backs:

- published Parquet datasets
- dataset manifests and contracts
- MLflow artifacts
- Airflow logs

It does not back a cluster-shared writable DuckDB warehouse file.

Any DuckDB database used by dbt or batch jobs is local or transient scratch and should
live on `emptyDir` or node-local `ReadWriteOnce` storage, not RWX NFS.

Configure the NFS export with `mapall` permissions so non-root pods can write:

- `mapall user`: `root`
- `mapall group`: `wheel`

Recommended dataset permissions:

- Owner: `root`
- Group: `wheel` or `root`
- Mode: `770`

The share is mounted on all k3s nodes at `/mnt/truenas` by the `nfs_client` Ansible
role and exposed inside the cluster via `k8s/nfs-pv.yml`.

NFS server IP and export path are configured in
`ansible/inventory/group_vars/k3s_servers.yml` and must match `k8s/nfs-pv.yml`.

### Ansible Inventory

`ansible/inventory/hosts.ini` is auto-generated by `tofu apply` via the
`ansible/ansible` provider. Do not edit it manually.

## Validation and CI

Infrastructure changes are validated through both local hooks and shared CI:

| Path | Scope | Command or file |
|---|---|---|
| Local hooks | Generic file hygiene | `.pre-commit-config.yaml` via `pre-commit` |
| Local hooks | Ansible syntax, YAML, and lint | `make -C infra ansible-lint` |
| Local hooks | Kubernetes manifest YAML lint | `make -C infra yaml-lint-k8s` |
| Local hooks | OpenTofu formatting | `make -C infra tf-fmt` |
| Local hooks | OpenTofu format + validate | `make -C infra tf-lint` |
| GitHub Actions | Shared infra validation | `.github/workflows/infra-checks.yml` |

The current workflow fans out into three jobs on the self-hosted runner fleet:

- `Ansible`: installs collections and runs `make ansible-lint`
- `Terraform`: runs `make tf-lint`
- `Kubernetes and Helm`: runs `make yaml-lint-k8s`, `make helm-lint`, `make helm-render`, and the monitoring wiring checks mirrored by `make validate-monitoring`

To enable local hooks:

```bash
make pre-commit-install
make pre-commit-run
```

## Layer Details

### Layer 1 - OpenTofu (`terraform/proxmox/`)

Provisions the three Debian 13 k3s VMs plus the external PostgreSQL VM with static
IPs, SSH keys, and hostnames. See `terraform/proxmox/README.md` for variables and
outputs.

This layer also writes local helper files consumed later in the workflow, including the
generated Ansible inventory, generated PostgreSQL host vars under
`ansible/inventory/host_vars/`, `k8s/.airflow-db.env`, `k8s/.ml-db.env`, and
`k8s/.grafana-admin.env` for Grafana admin secret bootstrapping.

```bash
cd terraform/proxmox
cp terraform.tfvars.example terraform.tfvars
tofu init
tofu apply
```

### Layer 2 - Ansible (`ansible/`)

Bootstraps k3s, configures the external PostgreSQL VM, installs NFS client utilities,
verifies shared storage connectivity, and copies kubeconfig locally when
`k3s_kubeconfig_local` is enabled.

If you enable etcd snapshot replication to S3-compatible storage, create encrypted
vault file `ansible/inventory/vault/k3s_etcd_snapshot_s3.yml` before running
Ansible. Non-secret toggles and overrides stay in
`ansible/inventory/group_vars/k3s_servers.yml`.

```bash
cd ansible
ansible-galaxy collection install -r requirements.yml
ansible-playbook bootstrap.yml
ansible-playbook playbooks/postgresql.yml
# or, when etcd S3 backup is enabled:
ansible-playbook bootstrap.yml --ask-vault-pass
```

### Layer 3 - kubectl + Helm (`k8s/` + `helm/`)

The checked-in manifests and values files currently support:

- shared namespaces and NFS persistent volumes
- VictoriaMetrics in `monitoring`
- Grafana in `monitoring`
- Grafana sidecar dashboard ConfigMaps generated from `k8s/monitoring/dashboards/`

Deploy the current stack with:

```bash
cp k8s/.airflow-app.env.example k8s/.airflow-app.env
cp k8s/.ml-app.env.example k8s/.ml-app.env
cp k8s/.monitoring-app.env.example k8s/.monitoring-app.env
# edit only the env files you actually need
kubectl apply -f k8s/namespaces.yml
kubectl apply -f k8s/nfs-pv.yml
make apply-secrets
make render-monitoring-manifests
make deploy-monitoring
```

Local secret inputs are split by source:

- `k8s/.grafana-admin.env` is generated by OpenTofu and used to create
  `grafana-admin-credentials` in the `monitoring` namespace.
- `k8s/.airflow-db.env` is generated by OpenTofu and contains the PostgreSQL
  connection string for `airflow-app-secrets`.
- `k8s/.airflow-app.env` is user-managed, gitignored, and contains the non-database
  Airflow keys merged into `airflow-app-secrets` in `airflow`.
- `k8s/.ml-db.env` is generated by OpenTofu and contains the MLflow backend store URI
  for `ml-app-secrets`.
- `k8s/.ml-app.env` is user-managed, gitignored, and contains the non-database ML app
  keys merged into `ml-app-secrets` in `ml`.
- `k8s/.monitoring-app.env` is user-managed, gitignored, and maps directly to
  `monitoring-app-secrets` in `monitoring`.
- Each manual env file has a committed `.example` template for new contributors.

`make apply-secrets` applies the generated Grafana secret and merges any generated DB
env files with manual app env files whose local files exist. Airflow additionally
requires `k8s/.airflow-app.env` plus either generated `k8s/.airflow-db.env` or a
manual `connection=` entry in `k8s/.airflow-app.env`.

The Airflow metadata database architecture is documented in
`docs/adr/adr-018-airflow-external-postgresql-metadata.md`: Airflow uses an external
PostgreSQL server running on its own Proxmox VM rather than the chart's embedded
PostgreSQL dependency or another in-cluster database service.

Connection strings and related credentials should stay in Kubernetes Secrets rather
than plaintext values files. Airflow should read the external PostgreSQL connection from
`airflow-app-secrets`, and MLflow should read its backend store URI from
`ml-app-secrets`. PgBouncer is tracked separately in deferred ADR-019 rather than as
part of the currently accepted baseline.

MLflow targets exist in `infra/Makefile`, but a full checked-in MLflow application
deployment is not in place yet. `make deploy-mlflow` currently skips when
`helm/mlflow.yml` is not present.

Monitoring persistence for Grafana and VictoriaMetrics stays on local-path storage
rather than NFS.

The checked-in Grafana dashboards are intentionally limited to assets with clear
upstream licensing:

- dashboard 315, Kubernetes cluster monitoring via Prometheus (MIT)
- dashboard 10229, VictoriaMetrics single-node (Apache-2.0)

Vendored dashboard JSON lives under `k8s/monitoring/dashboards/`. `kubectl`
Kustomize renders those files into dashboard ConfigMaps via
`k8s/monitoring/kustomization.yaml`, `make render-monitoring-manifests` writes
the generated JSON file to `k8s/monitoring/rendered/grafana-dashboards.json`,
and `make deploy-monitoring` applies the same Kustomize source with
`kubectl apply --server-side -k` so large dashboard ConfigMaps do not exceed
client-side apply annotation limits. The rendered JSON is a generated local
artifact, while
scoped upstream notices live under `third_party/grafana-dashboards/` so the
repo is not relicensed as MIT or Apache-2.0 as a whole.

## Sensitive Files

These files are gitignored and must not be committed:

| File | Contents |
|---|---|
| `terraform/proxmox/terraform.tfvars` | Proxmox API token, SSH key paths |
| `terraform/proxmox/terraform.tfstate*` | OpenTofu state |
| `ansible/inventory/hosts.ini` | Generated inventory |
| `ansible/inventory/host_vars/*.yml` | Generated PostgreSQL application credentials |
| `ansible/inventory/k3s_cluster.key` | Cluster join token |
| `ansible/inventory/vault/*.yml` | Vaulted Ansible secrets such as etcd S3 snapshot config |
| `k8s/.airflow-db.env` | Generated Airflow database connection string |
| `k8s/.airflow-app.env` | Manual Airflow non-database secrets |
| `k8s/.ml-db.env` | Generated MLflow backend store URI |
| `k8s/.ml-app.env` | Manual ML application non-database secrets |
| `k8s/.monitoring-app.env` | Manual monitoring application secrets |
| `k8s/.grafana-admin.env` | Grafana admin credentials |
