# infra/

Infrastructure-as-code for the homelab deployment of the eve-market-analytics
platform. The checked-in repo currently covers Proxmox VM provisioning, Ansible-based
k3s bootstrap, shared NFS storage wiring, and the monitoring stack.

## Directory Structure

```text
infra/
├── Makefile                    # Single entrypoint for infrastructure workflows
├── terraform/
│   └── proxmox/                # Layer 1: VM provisioning + inventory generation
├── ansible/                    # Layer 2: OS config + k3s bootstrap + NFS client setup
├── helm/                       # Layer 3: Helm values overrides committed today
│   ├── grafana.yml
│   └── victoriametrics.yml
└── k8s/                        # Layer 3: Raw manifests (namespaces, NFS PersistentVolumes)
```

## Bootstrap Order

Apply infrastructure in this sequence:

```text
1. OpenTofu     (terraform/proxmox) -> VMs created, inventory generated
2. Ansible      (ansible/)          -> k3s bootstrapped, NFS mounted, kubeconfig copied
3. kubectl/Helm (k8s/ + helm/)      -> base resources and monitoring services deployed
```

`make bootstrap` exists for the eventual full stack, but the currently committed Helm
values only cover Grafana and VictoriaMetrics. For the current repo state, use the
layered workflow below instead of assuming every application service is ready to deploy.

## Prerequisites

### Workstation Tools

Install tools via `mise`:

```bash
mise install
```

Required tools:

| Tool | Purpose |
|---|---|
| `tofu` (>= 1.11) | VM provisioning |
| `ansible-playbook` | Cluster configuration |
| `helm` | Helm chart rendering and deployment |
| `kubectl` | Cluster interaction |

### Proxmox Cluster

1. Create a Proxmox API token at Datacenter -> Permissions -> API Tokens.
2. Prepare a Debian 13 cloud-init template.
3. Use a shared datastore that supports imports and snippets.
4. Ensure `vmbr0` or your chosen bridge is available.
5. Keep SSH access available for snippet upload.

### TrueNAS NFS Share

Shared NFS storage backs:

- published Parquet datasets
- dataset manifests and contracts
- MLflow artifacts
- Airflow logs

It does not back a cluster-shared writable DuckDB warehouse file.

Any DuckDB database used by dbt or batch jobs is local or transient scratch and should
live on `emptyDir` or node-local `ReadWriteOnce` storage, not RWX NFS.

Configure the NFS export with `mapall` permissions so non-root pods can write:

- `mapall user`: `root`
- `mapall group`: `wheel`

Recommended dataset permissions:

- Owner: `root`
- Group: `wheel` or `root`
- Mode: `770`

The share is mounted on all k3s nodes at `/mnt/truenas` by the `nfs_client` Ansible
role and exposed inside the cluster via `k8s/nfs-pv.yml`.

NFS server IP and export path are configured in
`ansible/inventory/group_vars/k3s_servers.yml` and must match `k8s/nfs-pv.yml`.

### Ansible Inventory

`ansible/inventory/hosts.ini` is auto-generated by `tofu apply` via the
`ansible/ansible` provider. Do not edit it manually.

## Layer Details

### Layer 1 - OpenTofu (`terraform/proxmox/`)

Provisions three Debian 13 VMs with static IPs, SSH keys, and hostnames. See
`terraform/proxmox/README.md` for variables and outputs.

This layer also writes local helper files consumed later in the workflow, including the
generated Ansible inventory and `.grafana-admin.env` for Grafana admin secret
bootstrapping.

```bash
cd terraform/proxmox
cp terraform.tfvars.example terraform.tfvars
tofu init
tofu apply
```

### Layer 2 - Ansible (`ansible/`)

Bootstraps k3s, installs NFS client utilities, verifies shared storage connectivity,
and copies kubeconfig locally when `k3s_kubeconfig_local` is enabled.

If you enable etcd snapshot replication to S3-compatible storage, create encrypted
vault file `ansible/inventory/vault/k3s_etcd_snapshot_s3.yml` before running
Ansible. Non-secret toggles and overrides stay in
`ansible/inventory/group_vars/k3s_servers.yml`.

```bash
cd ansible
ansible-galaxy collection install -r requirements.yml
ansible-playbook bootstrap.yml
# or, when etcd S3 backup is enabled:
ansible-playbook bootstrap.yml --ask-vault-pass
```

### Layer 3 - kubectl + Helm (`k8s/` + `helm/`)

The checked-in manifests and values files currently support:

- shared namespaces and NFS persistent volumes
- VictoriaMetrics in `monitoring`
- Grafana in `monitoring`

Deploy the current stack with:

```bash
kubectl apply -f k8s/namespaces.yml
kubectl apply -f k8s/nfs-pv.yml
make deploy-monitoring
```

`make deploy-monitoring` expects `.grafana-admin.env` to exist locally so it can create
the `grafana-admin-credentials` Secret in the `monitoring` namespace.

Airflow and MLflow targets exist in `infra/Makefile`, but their Helm values files are
not committed in this repo yet.

Monitoring persistence for Grafana and VictoriaMetrics stays on local-path storage
rather than NFS.

## Sensitive Files

These files are gitignored and must not be committed:

| File | Contents |
|---|---|
| `terraform/proxmox/terraform.tfvars` | Proxmox API token, SSH key paths |
| `terraform/proxmox/terraform.tfstate*` | OpenTofu state |
| `ansible/inventory/hosts.ini` | Generated inventory |
| `ansible/inventory/k3s_cluster.key` | Cluster join token |
| `ansible/inventory/vault/*.yml` | Vaulted Ansible secrets such as etcd S3 snapshot config |
| `.grafana-admin.env` | Grafana admin credentials |
